For Developer:

Automated Manifest Generation:

Our API now automates the generation of VPM repository manifests directly from your package.json file. This eliminates the need for you to manually configure your VPM repository. You only need to release zip file in your repository, with no requirement for continuous integration (CI).

One VPM Repository, One GitHub Repository:

This principle simplifies the management of your VPM repositories. You can define custom IDs for your packages, as opposed to using the default format of io.github.owner.repo.

For User:

One VPM Repository, One GitHub Repository:

You can install VPM packages with precision, selecting only the packages you require without any unnecessary inclusions. Dependencies need to be installed separately.
